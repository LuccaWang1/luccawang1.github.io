How I Built This Website

While developing this website, I employed my software development, problem-solving, and communication skills. Through technical writing and documentation of my development process, I'm able to provide insight into the website's construction. I appreciate your curiosity of it. (You can find more documentation I've written on my other projects on GitHub here and here.) In this documentation, I alternate between past and present tense, because I built this in the past but maintain it in the present.

  1. Authorship: First off, I hand coded every character of this site. I didn't use a template, and it's not hosted anywhere other than GitHub - I built all of this myself. (I'm not opposed to templates, and I have experience building sites on hosting platforms, but this was a really fun challenge to showcase my skills.) Although I respect AI tools and use them for some questions and the ocassional light debugging (and am exploring GenAI, AI, and ML), the code came from me, Lucca Wang -- not the ever popular ChatGPT or any other source.
  2. Planning: I engaged in strategic planning and thorough research, asking myself, "What information do I need to include on this portfolio website?" Maintaining a modern web design while presenting key details in a way that reflects who I am is important to me.

  3. Technologies: To launch my portfolio quickly, I built a static website and deployed through GitHub Pages. (Being able to work quickly demonstrates my ability to produce quality products under pressure and within time constraints, as shown by my track record.) I developed and maintain this website with HTML Living Standard (which is The Latest version of HTML, beyond HTML5), CSS, and VSCode as my development environment, with Git for version control. I hosted the project's repository on GitHub and deployed the site via GitHub Pages. I integrated Google Fonts API to enhance the type. And, I implemented Open Graph protocol to ensure the portfolio displays rich previews that are engaging and polished when the website is shared on social media platforms.

    Example of Problem Solving: In developing the site, I explored and implemented solutions within the constraints of a static website built with HTML, CSS, and JavaScript, deployed via the GitHub Pages. Rather than setting up server-side routes for templates, as I might with Python, I created separate HTML files for each page and linked them directly. This method allows for seamless navigation within the static structure.

    The navigation bar mimics conditional rendering, enhancing the user experience (UX) as users move between pages. This approach not only adheres to GitHub Pages' requirements but also streamlines navigation and conserves space on the always-visible navigation bar, ensuring direct access to all site content while also prioritizing user interface (UI) design.

    Currently, I designed the navigation bar to display in a single line on larger screens, and to wrap into two lines on mobile devices. I've been considering the implications of non-conditional rendering.

  4. Domain: I selected a domain provider and registered a domain name, then configured it by updating the CNAME file.
  5. Assets: I gathered and organized my assets, including a PDF file and various images - primarily PNG files, as they are well-suited for web use. I utilized GIF images. The HTML5 update in 2014 to the present HTML Living Standard enables me to implement audio directly on the site. I took care with naming conventions and the directory structure. I customized images via icons8 for the favicon and for the social media/contact icons.
  6. Webpages: I built out index (the homepage) with HTML. Then, I added more webpages, like this one. I prioritize a thoughtful coding style that emphasizes readability and maintainability. I use alt and title attributes, along with section tags, to enchance accessibility.
  7. Styling: I ensured the website was consistent and accessible across both computer monitors and mobile screens by implementing responsive design with CSS media queries. I adhered to the Americans with Disabilities Act (ADA) Guidelines for font size and applied strong design principles, such as hierarchy. (ADA Guidelines also closely align with the Web Content Accessibility Guidelines (WCAG).) I enjoyed working with CSS selectors, especially leveraging powerful classes and as-is DOM elements. The animations add a dynamic touch. I utilized CSS variables (custom properties) for the colors in the stylesheet (instead of hex color codes) for reusability and theming. The variables are great for maintenance and readability.
  8. Learning: I implemented a new-to-me technology, Facebook's Open Graph. I opened the can of GitHub capabilities, and dove into advanced CSS. Overall, I also deepened my understanding of the website's construction, especially after employing documentation of it's creation.
  9. Gratitude: Thank you for visiting! I'm grateful for opportunity to show you my work and skills.
