How I Built This Website

While developing this website, I employed my software development, problem-solving, and communication skills. Through technical writing and documentation of my development process, I'm able to provide insight into the website's construction. I appreciate your curiosity of it. (You can find more documentation I've written on my other projects on GitHub here and here.)

  1. Authorship: First off, I hand coded each character of this site. The code came from me, Lucca Wang -- not the ever popular ChatGPT or somewhere else.
  2. Planning: I engaged in strategic planning and thorough research, asking myself, "What information do I need to include on this portfolio website?" Maintaining a modern web design while presenting key details in a way that reflects who I am is important to me.

  3. Stack and technologies: To launch my portfolio quickly, I used HTML Living Standard, CSS, and VSCode as my development environment, with Git for version control. I hosted the project's repository on GitHub and deployed the site via GitHub Pages. I integrated Google Fonts API to enhance the type. And, I implemented Open Graph protocol to ensure the portfolio displays rich previews that are engaging and polished when the website is shared on social media platforms.

    (Example of problem solving:) As I developed the site, I explored and implemented solutions within the parameters of this setup. The site is a static website with only HTML, CSS, and JavaScript supported via the GitHub Pages deployment. Instead of setting up routes and using server-side rendering for those templates, as I might with Python, I manually created separate HTML files for each page and linked them directly. This approach allows users to navigate between pages seamlessly within the static structure.

  4. Domain: I selected a domain provider and registered a domain name, then configured it by updating the CNAME file.
  5. Asset gathering: I gathered and organized my assets, including a PDF file and various images - primarily PNG files, as they are well-suited for web use. I took care with naming conventions and the directory structure. I customized images via icons8 for the favicon and for the social media/contact icons.
  6. Webpages: I built out index (the homepage) with HTML. Then, I added more webpages, like this one. I've refined my coding style, which I find to be readable and elegant. I'm a fan of the alt and title attributes and section tags to enchance accessibility.
  7. Styling: I ensured the website was consistent and accessible across both computer monitors and mobile screens by implementing responsive design with CSS media queries. I adhered to ADA guidelines for font size and applied strong design principles, such as typographic hierarchy. I enjoyed working with CSS selectors, especially leveraging classes and DOM elements. The animations add a dynamic touch.
  8. Learning: I implemented a new-to-me technology, Facebook's Open Graph, and I increased my knowledge on GitHub and CSS skills. Overall, I also deepened my understanding of the website's construction, especially after employing documentation of it's creation.
  9. Gratitude: Thank you for visiting! I'm grateful for opportunity to show you my work and skills.
